Программа для просмотра и поиска в больших файлах и сырых дисках. 
Предназначена для анализа бинарных данных, поиска криптографических ключей, паттернов wallet.dat, 
Electrum-сидов и пользовательских регулярных выражений.

✨ Возможности
Просмотр файлов
📄 Открытие файлов любого размера (потоковое чтение)
💾 Прямой доступ к сырым дискам через \\.\X:
🔢 Hex-дамп: 25 строк × 16 байт с ASCII-представлением
🧭 Навигация: PgUp/PgDn для перелистывания страниц
Поиск ---------------------------------------------------------------------------------------------------------
🔍 wallet.dat: поиск маркера \x00\x62\x31\x05 + ключевых слов key/mkey/wkey в радиусе 28 992 байт
🔍 Electrum: поиск Base64-последовательностей по паттерну (?<![a-zA-Z0-9/+])Q(?=.*\+)[a-zA-Z0-9+/=]{1000,100000} с валидацией
🔍 ciphertext: регистронезависимый поиск слова ciphertext
🔍 Свой паттерн: пользовательские регулярные выражения с историей (10 последних)
--------------Дополнительно
⏸️ Пауза поиска по F3 (для wallet, ciphertext, custom)
📊 Индикация прогресса в реальном времени
⌨️ Прерывание поиска по Ctrl+C
💾 Автосохранение найденных Electrum-данных в папку findet/
🧹 Валидация данных: отсечение мусора по =, /// и частоте символов
🔁 История паттернов в search_patterns.ini
--------------🛠 Требования
ОС: Windows 7/8/10/11
Python: 3.6 или выше
Зависимости: только стандартная библиотека (tkinter, re, threading, configparser)
-------------------------------------------------------------------------------------------------------------------
🔍 Режимы поиска
------------------wallet.dat
Ищет байтовую последовательность \x00\x62\x31\x05
При нахождении сканирует вперёд на 28 992 байта (0x7140)
Ищет первое вхождение key, mkey или wkey
Выводит 3 строки дампа (начиная со строки перед найденным ключом) в нижнюю панель
Останавливается и ждёт нажатия F3

----------------- Electrum
Ищет Base64-подобные последовательности длиной 1000–100000 символов
Валидация перед сохранением:
Не более 2 знаков =
Не более 2 подряд идущих /
Ни один символ не встречается чаще 1/3 от длины строки
Сохраняет прошедшие валидацию данные в findet/electrum_001.txt, electrum_002.txt и т.д.
Не останавливается — поиск продолжается автоматически

-------------- ciphertext
Регистронезависимый поиск слова ciphertext
При нахождении показывает смещение и останавливается
Ждёт нажатия F3 для продолжения

----------------Свой паттерн
Открывается диалог с полем ввода и историей (10 последних паттернов)
Паттерн компилируется как байтовое регулярное выражение (re.compile(pattern.encode('utf-8')))
При нахождении — пауза и ожидание F3
История сохраняется в search_patterns.ini

⚙️ Конфигурация
Файл search_patterns.ini создаётся автоматически при первом использовании «Своего паттерна».

Паттерны разделяются строкой |||
Максимум 10 записей (старые удаляются автоматически)
Кодировка: UTF-8

⚠️ Важные замечания
Работа с дисками
Для доступа к \\.\X: требуется запуск от имени Администратора
Некоторые сектора могут быть нечитаемы (системные, занятые другими процессами)
При ошибке чтения программа показывает предупреждение и откатывается на одну страницу
Размер диска может не определяться (os.path.getsize возвращает ошибку) — в этом случае поиск идёт до конца чтения
Производительность
Чтение блоками по 1 МБ с перекрытием 100 КБ для надёжного поиска на границах
Выравнивание позиций чтения по 1024 байта (для дисков — дополнительно по 512 байта)
Обновление статуса прогресса — раз в 10 МБ, чтобы не перегружать GUI
Безопасность
Программа только читает данные, не вносит изменений в файлы или диски
Найденные Electrum-данные сохраняются локально в папку findet/ — проверяйте содержимое перед использованием
